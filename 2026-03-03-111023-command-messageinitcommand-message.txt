  What I learned:

  MLOps in 2026 has split into two parallel tracks: classic MLOps and LLMOps —
  and the community is actively debating whether they converge or stay separate.
   The loudest signal from X is that people who skip MLOps fundamentals and jump
   straight to agents fail, per @techNmak (1,067 likes). The practical
  consensus: learn foundations first, then layer on LLM-specific tooling.

  The "production-ready" stack has crystallized around a core set of open-source
   tools. @kmeanskaran's 2-part guide on deploying MLOps + Agentic AI on AWS
  (898 likes) reflects the dominant pattern: Docker + EKS (Kubernetes) +
  Terraform + GitHub Actions CI/CD, with MLflow or W&B for tracking. FastAPI +
  Redis + Prometheus for serving and monitoring. This isn't aspirational — it's
  what people are actually shipping with.

  LLMOps adds three new layers on top of traditional MLOps: prompt/dataset
  versioning, evaluation & guardrails (hallucination/toxicity/bias detection),
  and trace-level observability for agent tool calls. Langfuse (open-source) and
   Databricks Mosaic AI are the most-discussed platforms for this, per multiple
  web sources. The primary bottleneck in 2026 isn't building a prototype — it's
  proving the prototype is safe for production.

  Agent-based AI systems are the frontier. Production agents need a reliability
  layer: strict read/write boundaries, memory discipline, guardrails/evals, and
  tracing. The pattern is plan → call tools → verify outcomes. Only 17% of
  enterprises have formal AI governance, which is the biggest gap, per ZenML and
   OneReach.

  KEY PATTERNS from the research:

  1. MLflow is the de facto standard — experiment tracking, model registry, and
  now GenAI tracing via OpenTelemetry, per multiple sources across web and X
  2. Kubernetes-native everything — Kubeflow for orchestration, KServe for
  serving, containerized pipelines are table stakes, per @kmeanskaran
  3. Observability > monitoring — drift detection alone isn't enough; you need
  distributed tracing across LLM chains, RAG retrievals, and agent tool calls,
  per Langfuse and Evidently docs
  4. Policy-as-code governance — automated fairness, compliance, and data
  lineage baked into CI/CD, per KDnuggets
  5. Start with the graviraja/MLOps-Basics repo — week-by-week curriculum
  covering W&B, DVC, Docker, CI/CD, model packaging — the most recommended
  learning resource, per @techwith_ram (534 likes)
